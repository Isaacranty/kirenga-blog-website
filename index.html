Before You Reach Out

Real Questions from Real Clients

These are the questions I actually get asked before someone decides to work with me.

How do I know you'll actually deliver? I've been burned before. +
That's the right question to ask. Here's what protects you: I give a written scope and timeline before any money changes hands. You pay 50% upfront and 50% only on delivery — so you never pay for work you haven't seen. I've delivered 30+ projects without a missed deadline, and every client above can be contacted directly for a reference.
Your prices seem high for Uganda. Why should I pay more than a cheaper option? +
Cheaper options exist. I've also fixed their work on more than a few occasions — at higher cost than doing it right the first time. What you're paying for is reliability, documentation, post-delivery support, and someone who answers their phone when something breaks at midnight. If budget is a real constraint, tell me — I'll tell you honestly whether I'm the right fit or not.
What happens after the project is delivered? Will you disappear? +
Every project includes a handover session, written documentation, and post-delivery support (7 to 90 days depending on package). After that, I'm available on a monthly retainer or ad-hoc basis. Most of my clients have been working with me for over a year because it's cheaper to maintain than to rebuild.
I'm not technical. Will I understand what you're doing? +
Yes — and that's something I take seriously. I won't hide behind jargon. Before every project I explain in plain language what I'm building, why, and what it means for your business. You'll receive updates at every milestone, and I won't consider a project done until you understand what you've received.
Can you work remotely? I'm not based in Kampala. +
Yes — most of my work is already done remotely via SSH, VPN, Zoom, and WhatsApp. I have active clients in Kenya, Rwanda, and internationally. For physical network installations, I can travel within Uganda. Remote-first is how I work.
How quickly can you start? +
If I have a slot open, I can begin within 48 hours of the deposit. The availability badge at the top of this page shows my current status. If I'm fully booked, I'll tell you the next available slot and we can schedule in advance. I never take on more than I can deliver properly.
What if I don't know exactly what I need? +
That's actually the most common situation. Send me a message describing your problem — not what you think the solution is, just the problem. I'll diagnose it, tell you what I'd recommend, and give you a clear scope before we agree on anything. No obligation, no cost for the consultation.

#!/bin/bash
# Server Hardening Script — Kirenga Isaac
# Ubuntu 22.04 LTS | kirengaisaac.github.io

set -e

# Update system
apt update && apt upgrade -y

# Configure UFW firewall
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

# Harden SSH
sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/#MaxAuthTries 6/MaxAuthTries 3/' /etc/ssh/sshd_config
systemctl restart sshd

# Install & configure fail2ban
apt install -y fail2ban
systemctl enable fail2ban
systemctl start fail2ban

echo "✅ Server hardening complete!"

# Backup Monitor — Kirenga Isaac
# Checks backup integrity & sends email alert

import os, smtplib, hashlib
from datetime import datetime
from email.mime.text import MIMEText

BACKUP_DIR = "/backup"
EMAIL      = "kirengaisaac@gmail.com"

def check_backup():
    today = datetime.now().strftime("%Y%m%d")
    backup = os.path.join(BACKUP_DIR, today)
    
    if not os.path.exists(backup):
        send_alert(f"❌ Backup MISSING for {today}")
        return False
    
    size = os.path.getsize(backup)
    if size < 1024:
        send_alert(f"⚠️ Backup too small: {size} bytes")
        return False
    
    print(f"✅ Backup OK: {size/1024/1024:.1f} MB")
    return True

if __name__ == "__main__":
    check_backup()
